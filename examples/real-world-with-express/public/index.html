agent-runtime · Express demo UI

This page is a browser front-end for the example Express server: it calls the same JSON routes as any API client. The engine runs an agent loop (LLM → parsed steps → optional tools) and can pause in a wait state until you resume with new text. Nothing here is “chat streaming” from the model token-by-token unless you add that separately — the SSE section only streams engine hook events (thoughts, actions, observations).

Vanilla HTML/JS, same origin as the API. Optional API_KEY in .env: paste the bearer token below (saved in sessionStorage for this tab only).

Chat POST /v1/chat

Sends one user message to the chat agent and waits until the run finishes (completed, failed, or waiting). The JSON body includes your Session ID so the run is stored under that session. Response contains reply (final result text from the protocol).

Chat stream (SSE hooks) POST /v1/chat/stream

Same message as Chat, but the server responds with Server-Sent Events: each event is a hook from the run (e.g. a thought or action step, an observation after a tool). That is not raw LLM token streaming — it is structured steps from the agent runtime. Use fetch + readable stream (as this page does), not browser EventSource (GET-only).

Wait / resume POST /v1/runs/wait-demo · …/resume

Wait-demo agent uses a scripted LLM: first response is wait (run saved as waiting in the in-memory store). You then Resume with text (e.g. your name); the run continues and returns a result. If this session already has a waiting wait-demo run, the server returns 409 — use Resume on that run or switch session. This models external approvals or human-in-the-loop steps, not a second chat thread.

Session runs GET /v1/sessions/:id/status

Lists all persisted runs for the current Session ID (chat + wait-demo). Each run shows userInput (first message), resumeInputs (text sent after each wait), reply, and a step timeline (history): wait / result / tool observations, plus a synthetic resume step so you see your resume text in order. Thin response checks ?light=1 to skip per-run history (smaller payload). Expand Raw JSON for the exact API response.
